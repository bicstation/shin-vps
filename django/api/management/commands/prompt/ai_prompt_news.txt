タイトル: {raw_title}

【執筆ルール：重要】
1. 1行目は記事タイトルのみを記載してください（「タイトル：」などの装飾・ラベルは一切禁止）。
2. タイトルの直後に [SUMMARY] タグと [/SUMMARY] タグの間に、重要ポイントを3つ「・」で始まる箇条書きで抽出してください。
3. 本文は必ず「## 」や「### 」の見出しを使い、複数のセクションに分けて構成してください。
4. 解説には、専門用語（アーキテクチャ、プロセスルール、TDP、ベンチマーク等）を適切に使いつつ、以下の要素を必ず含めてください。
   ・前世代モデルや競合製品との技術的・価格的な比較
   ・その製品が市場（自作PC市場やクリエイティブ市場）に与える影響
   ・どのようなユーザー（ゲーマー、配信者、エンジニア等）に最適なメリットがあるか
5. HTMLタグは使用せず、Markdown形式で出力してください（スクリプト側でHTMLへ変換します）。
6. 文末には必ず以下の形式で、記事に最適な「カテゴリ」と「タグ」を出力してください。

[CAT]カテゴリ名[/CAT]
[TAG]タグ1,タグ2,タグ3,タグ4,タグ5[/TAG]

【出力の禁止事項】
・<!DOCTYPE html> や <html> などのHTML構造タグは絶対に出力しないでください。
・「はい、承知しました」などの返答や解説、```markdown などの囲み記号は一切不要です。
・記事本文の文字数は2000文字以上の圧倒的な情報量を目指してください。
