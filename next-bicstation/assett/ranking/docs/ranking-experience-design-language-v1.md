# Ranking Experience Design Language V1

Version: 1.0

Status: Approved Design Language

---

# Purpose

This document defines the official visual language of the SHIN CORE LINX Ranking Experience.

Its purpose is not to specify implementation details.

Its purpose is to ensure that every generated image, UI component, and frontend implementation expresses one consistent customer experience.

This document serves as the canonical design reference for the Ranking Experience.

---

# Design Philosophy

The Ranking Experience is not a product catalog.

It is not a specification database.

It is not a shopping mall.

The Ranking Experience is a premium editorial destination where customers compare products with confidence.

The visual language should communicate trust, quality, and clarity before information density.

---

# Design Principle

Reality

↓

Meaning

↓

Experience

↓

Decision

Every visual element should reduce customer uncertainty.

---

# Experience Goal

The customer should immediately understand:

• This is the official Ranking Experience.
• The best products are presented first.
• Navigation is simple and confidence-inspiring.
• Product comparison feels effortless.

---

# Visual Identity

Luxury Technology

Modern Japanese Web Design

Editorial Presentation

Premium Dark Theme

Elegant Minimalism

Professional Product Showcase

# Color Language

Primary

Deep Navy

Dark Blue

Soft Black

Secondary

White

Light Gray

Accent

Warm Gold

Gold represents importance.

It is never used excessively.

Blue represents semantic navigation and interaction.

---

# Hero

Purpose

Introduce the Ranking Experience.

Characteristics

Large cinematic background

Minimal copy

Premium atmosphere

Strong visual impact

Never overloaded.

---

# Featured Overall Ranking

Purpose

Present the best overall recommendation.

Characteristics

Largest secondary visual.

Gold accent.

Editorial presentation.

Highest trust.

---

# Semantic Navigation

Purpose

Express the shared Semantic Universe.

Navigation should organize Reality.

It must never redefine Reality.

Categories originate from the Backend Semantic Universe.

---

# Ranking Sections

Purpose

Group Ranking destinations.

Each section represents one Semantic Group.

Examples

Usage

Device

CPU

GPU

Memory

Storage

Monitor

Each section should feel like a chapter of a premium magazine.

---

# Ranking Cards

Purpose

Present a Ranking destination.

Cards should emphasize comparison rather than exploration.

Important information

Ranking Name

Short Description

Product Count

Call to Action

Cards should feel clean and premium.

---

# Imagery

Photorealistic.

Premium product photography.

Elegant architectural lighting.

Luxury desk environments.

Premium showroom atmosphere.

No unnecessary visual effects.

No excessive RGB.

No cyberpunk.

No outer space.

No fantasy.

All generated images must follow this Design Language.

Every Hero.
