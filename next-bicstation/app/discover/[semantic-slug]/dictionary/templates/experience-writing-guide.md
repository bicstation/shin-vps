# Purpose

The Experience Dictionary is a Translation Dictionary.

Its responsibility is not to define Semantic Reality.

Its responsibility is to communicate Semantic Reality in language that customers immediately understand.

```
Semantic Dictionary

↓

Experience Dictionary

↓

Customer
```

Every section should help customers naturally understand the Semantic World.

---

# General Principles

## Reality First

Do not invent new Semantic meaning.

Translate the existing Semantic Reality.

---

## Customer Language

Write in language that ordinary customers naturally understand.

Avoid internal terminology whenever possible.

---

## Consistency

Use a consistent writing style across all Semantic Worlds.

The customer should feel that every Discover Experience belongs to the same platform.

---

# Hero

## Purpose

Introduce the Semantic World.

The customer should immediately understand:

> "What kind of world is this?"

---

## Title

Use the common customer-facing name.

Examples

```
AI PC

ゲーミングPC

ビジネスPC
```

---

## Catch Copy

One short sentence.

Focus on emotion and expectation.

Examples

```
生成AIと未来のワークスタイルを、もっと身近に。

ゲームの世界を、最高のパフォーマンスで楽しもう。

毎日の仕事を、もっと快適でスマートに。
```

---

## Description

Approximately 80–150 characters.

Explain:

- Who this world is for
- What customers can discover
- What value they receive

---

# About

## Purpose

Explain:

> "What is this world?"

Begin with:

```
〇〇は、
...
```

Explain the characteristics of the Semantic World.

Do not recommend products.

---

# Elements

## Purpose

Introduce the important characteristics of the Semantic World.

Recommended title

```
この世界を構成する主な要素
```

Recommended description

```
〇〇を支える代表的な技術や特徴をご紹介します。
```

Keywords should represent the Semantic World.

---

# Representative Products

## Purpose

Introduce representative examples.

Products should never become the main subject.

The Semantic World remains the primary subject.

Recommended title

```
この世界を代表する製品
```

Recommended description

```
この世界を代表する〇〇です。

性能や特徴を比較しながら、
自分に最適なモデルを見つけましょう。
```

---

# Related Worlds

## Purpose

Introduce neighboring Semantic Worlds.

Do not describe them as ordinary navigation.

Emphasize that they belong to the same Semantic Category.

Recommended title

```
同じ用途カテゴリーの世界
```

Recommended description

```
〇〇用途と同じカテゴリーに属する、
さまざまなセマンティックワールドを探索できます。
```

---

# Continue Discovery

## Purpose

Encourage continued exploration.

Recommended title

```
次の世界を探索する
```

Recommended description

```
あなたに合った用途は他にもあります。

新しいセマンティックワールドを探索し、
自分に最適なPCとの出会いを広げましょう。
```

Recommended button

```
他のセマンティックワールドを探索する
```

```
Hero
center top

About
center 30%

Elements
center

Products
center 70%

Related

Continue
bottom
```

The objective is to create one continuous Semantic World.

---

# Experience Assets

Each Semantic World should define:

- Icon
- Accent Color
- Background Image
- Background Position

These are Experience assets.

They do not define Semantic Reality.
