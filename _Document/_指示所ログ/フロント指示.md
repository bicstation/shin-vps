

■ semantic rendering layer 方針確認結果

■ 結論

frontend に
semantic rendering layer を正式導入する方針を推奨

---

■ 理由

semantic attribute は：

「表示用タグ」

ではなく

「意味データ」

として扱う段階へ移行しているため

---

semantic attribute
↓
renderer mapping
↓
UI component

■ 補足（重要）

semantic rendering layer は frontend 実装ですが、

semantic meaning 自体は
backend semantic engine を source of truth として扱う方針です。

frontend は：

semantic attribute
↓
rendering policy
↓
UI expression

を担当し、

意味定義（priority / usage / semantic role）は
backend 側を正とします。

---

■ 意図

frontend が独自semanticを持ち始めると：

backend semantic
≠
frontend semantic

となるリスクがあるため

semantic authority は backend に統一します。
